--- layout: home title: "How to prepare your Moodle installation for ef4m scripts" ---

Difficulty: Easy
Level: Teacher/Admin
Moodle version: 3+

It is very easy to set up Moodle to use ef4m. This is one of the important aspects of ef4m, to be easy to setup and use by any Moodle user that has teacher access.

Three steps for an ef4m ready Moodle

Step 1: Create a new course

If you are a teacher you will probably need to ask for an extra course to be created for you, where you will store all your ef4ms. You can also reuse a course you already have. It doesnâ€™t have to be a brand-new course space.

Step 2: Transform the course into an ef4m repository

We now need to transform the course into an ef4m repository. Ef4m repositories need to be fully accessible by all users that exist on your Moodle installation, without needing to enrol them to the course. To do this we need to change the permissions to the course so that all users can have access to it without being participants.

We are going to change the course permissions. Course permissions are "buried" within the course participants page.

Click on Participants.

From the dropdown menu, choose Permissions.

Wait for the page to load and in the Filter textbox type search for the permission moodle/course:view. When it appears, use the + button under "Roles with permission" and add "Authenticated user" to the list.

What this says in Moodleze is that all users that can log in to our Moodle site can view the course without needing to be in the participants list â€“ effectively making the course fully accessible to all our Moodle users.

Step 3: Add a folder activity in the course

This folder activity will hold your ef4m scripts. The folder can either be a visible or a stealth activity.

Congratulations, you have now prepared your Moodle installation to run ef4m scripts.
